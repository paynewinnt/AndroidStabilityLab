# AndroidStabilityLab 项目文件说明

本文档说明当前仓库主要工程文件的用途，口径以 `Android Stability Lab` 当前代码结构为准。

说明范围：

- 包含：源码、脚本、配置、文档、测试和资源文件。
- 排除：`runtime/`、`data/`、`__pycache__/`、`.pyc`、`.pyo`、`.DS_Store` 等运行产物或本地缓存。
- 新增稳定性业务默认沉淀到 `stability/`；`core/`、`utils/` 只保留兼容或通用辅助能力。

## 根目录

- `README.md` / `README-CN.md`：项目介绍、安装方法和 CLI/Web 稳定入口说明。
- `requirements.txt`：运行依赖，覆盖 CLI/Web、数据库、导出、监控和稳定性主链路。
- `requirements-dev.txt`：本地开发和测试依赖。
- `check_env.py`：本地环境检查脚本。
- `ruff.toml`：Ruff lint/format 配置。
- `pyrightconfig.json`：Pyright 配置。

## assets

- `assets/icons/`：图标资源，供 Web、报告或后续打包入口复用。
- `assets/desktop/`：桌面或打包相关资源。

## config

- `config/database.json`：数据库连接配置，支持默认 SQLite 和可选 MySQL。
- `config/monitoring.json`：监控采样、后端预设和常用 ADB 命令配置。
- `config/performance.json`：采集性能优化参数。
- `config/performance_risk_thresholds.json`：性能风险阈值配置。
- `config/stability_rules.json`：稳定性分析规则配置。
- `config/rule_review_policy.json`：规则评审准入策略。
- `config/rule_review_baseline_policy.json`：规则评审基线晋升策略。
- `config/rule_replay_golden_samples.json`：规则回放 golden samples 配置。

## core / utils

- `core/`：历史兼容入口和基础辅助模块，例如 ADB collector、配置、数据和性能监控辅助。
- `utils/`：通用导出、校验等工具。

新的业务服务、领域模型、持久化适配和 Web/API 能力应优先放入 `stability/`，不要继续扩展历史兼容目录。

## scripts

- `scripts/verify_v1_acceptance.sh`：V1 一键验收入口。
- `scripts/verify_cli_query_smoke.sh`：CLI 查询 smoke。
- `scripts/verify_web_portal_smoke.sh`：Web 门户 smoke。
- `scripts/verify_*_smoke.sh`：Monkey、冷启动、前后台、安装卸载、重启、待机唤醒等模板 smoke。
- `scripts/mock_im_receiver.py`：本地 IM/webhook mock receiver。

## tests

- `tests/README.md`：测试入口、测试分层和 smoke 使用口径。
- `tests/test_*.py`：单元测试和集成测试，覆盖 CLI、Web、任务、执行、设备、准入、规则治理、outbox、监控和无人值守等主链路。
- `tests/helpers/`：Web、CLI 和领域对象测试辅助。

## docs

- `docs/prd/`：产品需求和版本需求。
- `docs/计划/`：开发计划、版本计划和验收清单。
- `docs/方案/`：总体方案、子系统方案和架构优化建议。
- `docs/运维/`：IM/飞书真实链路联调 runbook、验收记录模板和联调记录。
- `docs/Android Stability Lab平台使用说明.md`：平台使用说明。
- `docs/Android Stability Lab未完成功能验收清单.md`：未完成功能验收矩阵。
- `docs/项目文件说明.md`：当前文件说明。

## runtime.example / runtime / data

- `runtime.example/`：脱敏运行态示例说明。
- `runtime/`：本地报告、快照、runner 状态、outbox、APK 和产物目录，已忽略。
- `data/`：本地 SQLite 数据库和 WAL/SHM，已忽略。

`runtime/` 和 `data/` 可能包含真实设备 ID、内网地址、webhook secret、APK、日志和报告，不作为源码维护对象。

## stability 顶层

- `stability/bootstrap.py`：运行时装配入口，负责组装内存版和持久化版 service bundle。
- `stability/time_utils.py`：项目统一时间工具。

## stability/app

应用服务层，承接用例和业务编排：

- `device_service.py`：设备注册、状态同步、设备池摘要和详情。
- `task_service.py`：任务定义创建、查询、更新和摘要输出。
- `run_execution_service.py` / `run_execution/`：Run 执行主链路、并发、重试、监控、issue、artifact 和报告串联。
- `run_history_service.py`：Run 历史查询和详情输出。
- `analysis_service.py` / `issue_fingerprint_governance_service.py` / `comparison_service.py` / `regression_service.py`：问题聚合、指纹治理、对比和回归判断。
- `performance_trend_service.py`：性能趋势与风险判断。
- `rule_*`：规则治理、回放、golden suite、评审报告和基线能力。
- `admission_case_*` / `quality_gate_service.py` / `release_submission_service.py`：准入、质量门禁、配置化门禁策略和提测请求。
- `integration_outbox*`：outbox 事件、投递、重试、receipt 和 dead-letter 运维。
- `collaboration*`：actor、identity、session、权限和协作动作。
- `unattended*`：无人值守任务、巡检、日报和周报。
- `platform_health_service.py` / `doctor_service.py`：平台自监控和本地诊断。

## stability/domain

领域层：

- `models.py`：设备、任务、Run、实例、Issue、Artifact 等核心模型。
- `enums.py`：领域枚举。
- `errors.py`：统一 `AppError` 和错误码。
- `value_objects.py`：值对象、ID 和时间辅助。
- `*_models.py`：准入、规则、快照、集成、协作、性能趋势等专项领域模型。

## stability/execution

- `plan.py`：执行计划模型。
- `state_machine.py`：执行状态机。
- `hooks.py`：执行生命周期 hook。

## stability/infrastructure

基础设施和外部适配：

- `adb/`：正式 ADB collector 和指标采集模块。
- `device_adapter.py`：设备发现适配器。
- `monitoring_*`：ADB、SoloX、Perfetto 和组合监控适配器。
- `persistence/`：SQLAlchemy 连接、模型和连接池。
- `rule_config.py`：规则配置加载。
- `command_runner.py`：统一命令执行抽象。
- `artifact_paths.py`：产物路径规划。

## stability/repositories

- `memory.py`：内存仓储，主要用于测试和本地 smoke。
- `sqlalchemy.py`：SQLAlchemy 持久化仓储。
- `mappers.py`：领域对象与持久化记录之间的映射器。

## stability/scenario

场景执行器：

- `registry.py`：模板 registry，统一模板 ID、参数 schema、指标能力和风险说明。
- `monkey.py`：Monkey 模板执行器。
- `cold_start_loop.py`：冷启动循环模板执行器。
- `device_cycle.py`：安装卸载、重启、待机唤醒等设备循环执行器。
- `custom_automation.py`：自定义自动化执行器。
- `tcp_recovery.py`：TCP 设备重连 helper。

## stability/artifact / stability/issue

- `artifact/collector.py`：issue 证据抓取器。
- `artifact/remote_capture.py`：远端设备证据抓取 mixin。
- `issue/detectors.py`：异常检测器，识别 crash、ANR、native crash、reboot、process exit、device offline、startup timeout、freeze、black screen、watchdog 等问题。

## stability/cli

- `__main__.py`：`python -m stability.cli` 启动入口。
- `task_create.py`：兼容门面，保持旧 import 路径稳定。
- `parser*.py`：CLI 命令注册。
- `handlers/`：CLI 命令处理器。
- `payloads*.py`：CLI 输出 payload 构造。

## stability/web

- `server.py`：标准库 HTTP server 启动。
- `application.py`：Web 应用装配，把 feature mixin、payload、actions 和响应边界组合成 `WebPortalApplication`。
- `manifest.py`：页面、读接口、写操作、说明和接口类型的唯一 manifest 来源。
- `payloads.py`：API manifest 和 OpenAPI 风格 payload 构造。
- `routes.py`：共用路由常量和 route helper。
- `responses.py`：HTML/JSON/file 响应边界。
- `request_context.py`：请求、身份和受信 header 上下文。
- `feature_registry.py`：GET/POST feature handler 分发。
- `features/*/routes.py`：各功能模块路由。
- `features/*/payload.py`：页面/API payload。
- `features/*/page.py`：页面渲染。
- `features/*/actions.py`：POST 动作处理。

页面/API 清单以 `stability/web/manifest.py`、`/json-api`、`/api/manifest` 为准，README 和测试文档不再维护重复的完整清单。
