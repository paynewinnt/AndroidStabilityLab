# AndroidStabilityLab 项目文件说明

本文档说明当前仓库主要工程文件的用途，口径以 `Android Stability Lab` 新项目为准。

说明范围：

- 包含：源码、脚本、配置、文档、测试、资源文件。
- 排除：`runtime/`、`data/`、`__pycache__/`、`.pyc`、`.pyo`、`.DS_Store` 等运行产物或缓存。
- 说明口径：只说明文件在当前项目中的职责，不记录历史来源。

## 根目录

- `README.md`: 项目介绍、安装方法和 CLI/Web 入口说明。
- `check_env.py`: 本地环境检查脚本，用于验证 Python、依赖包和数据库模块是否可用。
- `requirements.txt`: Python 依赖清单，覆盖 CLI/Web、数据库、导出、监控和稳定性主线相关依赖。

## assets

- `assets/icons/`: 项目图标资源，供 Web、报告或后续打包入口复用。

## config

- `config/database.json`: 数据库连接配置，支持默认 SQLite 和可选 MySQL。
- `config/monitoring.json`: 监控采样、后端预设和常用 ADB 命令配置。
- `config/performance.json`: 采集性能优化参数，如 ADB timeout、缓存大小和采样间隔。
- `config/performance_risk_thresholds.json`: 性能风险阈值配置。
- `config/rule_replay_golden_samples.json`: 规则回放 golden samples 配置。
- `config/rule_review_baseline_policy.json`: 规则评审基线晋升策略。
- `config/rule_review_policy.json`: 规则评审准入策略。
- `config/stability_rules.json`: 稳定性分析规则配置。

## core / database / utils

- `core/`: ADB、配置和平台自监控等基础设施辅助模块。
- `database/`: SQLAlchemy 连接、表模型、维护和查询辅助模块。
- `utils/`: 通用工具模块。

新增稳定性业务默认沉淀到 `stability/`，上述目录仅作为基础设施和通用能力补充。

## stability 顶层

- `stability/bootstrap.py`: 运行时装配入口，负责组装内存版和持久化版 service bundle。
- `stability/time_utils.py`: 项目统一时间工具。

## stability/app

- `stability/app/device_service.py`: 设备注册、状态同步、设备池摘要和详情。
- `stability/app/task_service.py`: 任务定义创建、查询、更新和摘要输出。
- `stability/app/run_execution_service.py`: Run 执行主链路，负责实例执行、并发、重试、监控、issue、artifact 和报告串联。
- `stability/app/run_history_service.py`: Run 历史查询和详情输出。
- `stability/app/analysis_service.py`: 问题聚合、Top Issue 和分析结果输出。
- `stability/app/performance_trend_service.py`: 性能趋势与风险判断。
- `stability/app/quality_gate_service.py`: 质量门禁评估。
- `stability/app/admission_case_service.py`: AdmissionCase 聚合、状态、协作和准入结论。
- `stability/app/integration_outbox_service.py`: outbox 事件、投递、重试、receipt 和 dead-letter 运维。
- `stability/app/collaboration_service.py`: actor、identity、session、权限和协作动作。
- `stability/app/unattended*`: 无人值守任务、巡检、日报和周报。

## stability/artifact

- `stability/artifact/collector.py`: issue 证据抓取器。
- `stability/artifact/remote_capture.py`: 远端设备证据抓取 mixin。
- `stability/artifact/command_runner.py`: 命令执行抽象。

## stability/cli

- `stability/cli/__main__.py`: `python -m stability.cli` 启动入口。
- `stability/cli/parser*.py`: CLI 命令注册。
- `stability/cli/handlers/`: CLI 命令处理器。
- `stability/cli/payloads*.py`: CLI 输出 payload 构造。

## stability/domain

- `stability/domain/enums.py`: 领域枚举。
- `stability/domain/models.py`: 设备、任务、Run、实例、Issue、Artifact 等核心模型。
- `stability/domain/admission_case_models.py`: AdmissionCase 领域模型。
- `stability/domain/quality_gate_models.py`: QualityGate 领域模型。
- `stability/domain/integration_models.py`: outbox、webhook、receipt 等集成模型。

## stability/execution

- `stability/execution/plan.py`: 执行计划模型。
- `stability/execution/state_machine.py`: 执行状态机。
- `stability/execution/hooks.py`: 执行生命周期 hook。

## stability/infrastructure

- `stability/infrastructure/device_adapter.py`: 设备发现适配器。
- `stability/infrastructure/monitoring_*`: ADB、SoloX、Perfetto 和组合监控适配器。
- `stability/infrastructure/persistence/`: 持久化连接和模型适配。
- `stability/infrastructure/rule_config.py`: 规则配置加载。

## stability/issue

- `stability/issue/detectors.py`: 异常检测器，识别 crash、ANR、native crash、reboot、process exit、device offline、startup timeout、freeze、black screen、watchdog 等问题。

## stability/repositories

- `stability/repositories/memory.py`: 内存仓储。
- `stability/repositories/sqlalchemy.py`: SQLAlchemy 持久化仓储。
- `stability/repositories/mappers.py`: 领域对象与持久化记录之间的映射器。

## stability/scenario

- `stability/scenario/monkey.py`: Monkey 模板执行器。
- `stability/scenario/cold_start_loop.py`: 冷启动循环模板执行器。
- `stability/scenario/device_cycle.py`: 安装卸载、重启、待机唤醒等设备循环执行器。
- `stability/scenario/custom_automation.py`: 自定义自动化执行器。
- `stability/scenario/tcp_recovery.py`: TCP 设备重连 helper。

## stability/web

- `stability/web/server.py`: Web server 启动。
- `stability/web/application.py`: Web 应用装配。
- `stability/web/routes.py`: 路由常量。
- `stability/web/pages_*.py`: 页面渲染。
- `stability/web/application_payload_*.py`: 页面/API payload 构造。
- `stability/web/application_actions_*.py`: POST 动作处理。
- `stability/web/application_identity.py`: Web 身份解析和审计边界。

## 说明

新增代码优先放入 `stability/`，新增文档优先放入 `docs/prd`、`docs/计划`、`docs/方案` 或 `docs/运维`。运行产物默认落入 `runtime/` 和 `data/`，不作为源码维护对象。
