# 飞书 IM 真实链路 2 小时与 24 小时验收记录模板

## 1. 使用说明

本模板用于记录飞书自定义机器人真实链路验收，覆盖 2 小时试运行和 24 小时长时联调。

记录原则：

- 不记录 `FEISHU_SIGNING_SECRET` 明文。
- 不记录完整 `FEISHU_WEBHOOK_URL` token，只保留脱敏 URL。
- 未跑满 24 小时不能写成真实链路完成。
- 本地 mock/smoke 通过只能写成平台侧能力通过，不能替代真实飞书验收。

## 3. Worker 命令

单轮 canary：

```bash
./.venv/bin/python -m stability.cli run-feishu-notify-worker \
  --webhook-name "Android Stability Lab" \
  --limit-per-webhook 30 \
  --interval-seconds 0 \
  --max-rounds 1 \
  --stop-when-idle
```

2 小时试运行：

```bash
./.venv/bin/python -m stability.cli run-feishu-notify-worker \
  --daemon \
  --webhook-name "Android Stability Lab" \
  --interval-seconds 60 \
  --limit-per-webhook 20 \
  --max-runtime-seconds 7200
```

24 小时长时联调：
